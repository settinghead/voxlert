# VoiceForge for OpenClaw

AI-cloned game characters announce session events, task completions, and prompts using contextual phrases generated by an LLM and spoken through a local TTS server.

## What it does

This hook translates OpenClaw events into VoiceForge notifications:

| OpenClaw Event | VoiceForge Event | Description |
|---|---|---|
| `command:stop` | Task complete | Agent finishes work — LLM-generated contextual phrase |
| `command:new` | Session start | New session begins — fallback phrase from voice pack |
| `command:reset` | Session start | Session reset — fallback phrase from voice pack |
| `message:received` | Prompt acknowledged | User sends a message — fallback phrase from voice pack |

## Requirements

- **VoiceForge** installed (`npm install -g @settinghead/voiceforge` and `voiceforge setup`, with OpenClaw selected in the hooks step)
- **Node.js 18+**
- **Chatterbox or Qwen TTS** running locally (or fallback phrases only)
- **LLM API key** (e.g. OpenRouter) in VoiceForge config for contextual phrases

## Configuration

VoiceForge config is at `~/.voiceforge/config.json` (or `voiceforge config path`). Use the `voiceforge` CLI to manage settings:

```bash
voiceforge config              # Show current config
voiceforge pack use <pack-id>  # Switch voice pack
voiceforge voice               # Interactive voice picker
```

See the [VoiceForge README](https://github.com/settinghead/voiceforge) for full documentation.
